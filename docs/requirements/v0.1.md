# Pulsefolio Requirements v0.1

## User stories

### US-001: Register and authenticate
**As a** user, **I want** to register and log in **so that** my portfolio data is private.
- Acceptance: Email/password register, login, JWT refresh, Sign in with Apple (iOS, phase 2)

### US-002: Define multi-asset portfolio
**As a** user, **I want** to set target allocations across asset classes **so that** the AI rebalances toward my plan.
- Acceptance: Stocks, ETFs, bonds, crypto, commodities, cash; weights sum to 100%

### US-003: View live dashboard
**As a** user, **I want** a dashboard with portfolio value, risk score, and allocation **so that** I feel the pulse of my portfolio.
- Acceptance: Animated value, pulse line, live indicator, asset class donut

### US-004: Receive AI recommendations
**As a** user, **I want** AI to recommend rebalances with risk/return impact **so that** I make informed paper trades.
- Acceptance: Structured output with rationale, confidence, riskImpact, expectedReturnImpact

### US-005: Approve trades manually
**As a** user, **I want** to approve or dismiss recommendations **so that** I stay in control.
- Acceptance: Pending queue, approve executes trade, dismiss logs rejection

### US-006: Enable auto-mode (cloud)
**As a** user, **I want** auto-mode to run 24/7 in the cloud **so that** trades execute without my device online.
- Acceptance: Worker polls auto-mode users, executes within guardrails, notifies user

### US-007: Risk guardrails
**As a** user, **I want** server-side risk limits **so that** the portfolio stays balanced.
- Acceptance: Max 25% single asset, 15% crypto cap, 5% cash min, risk score visible

### US-008: Theme customization
**As a** user, **I want** to pick a visual theme **so that** the app matches my preference.
- Acceptance: Midnight, Aurora, Paper, Terminal; syncs across web (iOS phase 2)

### US-009: Trade history and AI insights
**As a** user, **I want** full trade history and decision log **so that** I can audit AI behavior.
- Acceptance: Immutable decision log, link recommendations to trades

### US-010: Paper trading disclaimer
**As a** user, **I want** clear paper-trading labels **so that** I never confuse simulation with real money.
- Acceptance: PAPER TRADING badge on all trading views
